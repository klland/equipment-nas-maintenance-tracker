# NAS / 網路設備巡檢與異常追蹤系統

這是一個以家用 NAS、Router、Switch、UPS 與工作站為情境的設備巡檢前端工具。作品目標不是假裝全自動監控，而是把居家設備維護流程工程化：設備清單、管理頁連線探測、手動巡檢、異常處理、SOP 與資料匯出。

## 專案定位

設備工程師在實務上不只是排除異常，也需要記錄現象、初判原因、處理方式、復歸確認與後續預防。本專案用家中可接觸的 NAS / 網路設備模擬這個流程，展示如何把維護資訊結構化。

## 功能

- 設備清單：新增 NAS、Router、Switch、PC、UPS、AP
- 管理頁連線探測：嘗試由瀏覽器連到設備 HTTP/HTTPS 管理頁
- 巡檢表：記錄容量、溫度、備份狀態、硬碟/RAID 狀態與現場確認項目
- 異常追蹤：記錄現象、類型、優先級、根因、處理方式與狀態
- SOP：內建 NAS 無法登入、備份失敗、硬碟警告、Wi-Fi 不穩等排查流程
- Dashboard：統計設備數、需注意設備、未結案數、最近巡檢與異常狀態
- localStorage 保存資料
- JSON 匯入 / 匯出備份

## 能力邊界

純前端瀏覽器不能直接做以下事情：

- ICMP ping
- 讀取 NAS SMART / RAID 內部資料
- 讀取 UPS 電池健康
- 自動取得路由器 WAN 狀態
- 自動修改 NAS 或 Router 參數

因此本專案把這些項目標示為「手動確認」。使用者可以從 NAS 或 Router 管理頁確認後，再把結果記錄到巡檢表。

## 技術

- HTML
- CSS
- JavaScript
- localStorage
- FileReader / Blob 用於 JSON 匯入匯出

## 使用方式

直接用瀏覽器開啟 `index.html`。

若要減少 `file://` 限制，也可以在專案資料夾執行本機伺服器：

```bash
python3 -m http.server 8080
```

再開啟 `http://localhost:8080`。

## 履歷說法

以 NAS 與網路設備為情境，開發設備巡檢與異常追蹤前端工具，實作設備清單、管理頁連線探測、巡檢紀錄、異常處理單、SOP 流程與 JSON 匯入匯出，用來模擬設備工程中的維護紀錄、復歸確認與預防改善流程。

## 面試說明

這個作品不是完整 CMMS / MES，也不是自動監控系統。它是我用家中 NAS、路由器和 UPS 做的小型維護流程工具。重點是把設備工程常見的巡檢、異常、處理、結案與預防流程做成可以操作和保存紀錄的前端作品。

若延伸到實際產線，可加入資料庫、登入權限、MTTR / MTBF、停機原因 Pareto、保養週期提醒，並串接設備 log、PLC、SCADA、SNMP 或廠務系統 API。
